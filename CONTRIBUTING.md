# Contributing

Thanks for helping keep this list useful. The goal is **honest, practical** entries — not a link dump.

## What gets accepted

An entry must clear two bars:

1. **It replaces a real recurring task.** Not "this is cool," but "this removes a chore you do every week."
2. **It explains *how*.** Every entry needs a plain-language "how to use it" line. Tools without a concrete use get rejected.

Bonus points for a **recipe** (a short routine) or a **🔧 For builders** note showing how to wire it up.

## Format

Add tools to the right section's table:

```markdown
| **[Tool Name](https://homepage)** 🟢 | One-line "what it is" | One-line "how to use it everyday" |
```

Use the legend tags where they apply:

- 🟢 No code needed
- 🔧 Builder / self-host option
- 🆕 Shipped or majorly upgraded in the last ~6 months
- 💸 Paid / subscription
- 🔒 Strong privacy / local option

## Rules

- **Real links only.** Link to the official homepage. No referral/affiliate links.
- **Be honest about cost and gating.** If it's behind a pricey plan, tag it 💸 and say so.
- **No hype.** Describe what it actually does today, not the roadmap.
- **Frontier section is for genuine gaps** — things that don't exist or work poorly yet, with the opportunity spelled out.

## How to submit

1. Fork the repo
2. Add your entry in the appropriate section
3. Open a PR describing the recurring task it solves

Disagreements about inclusion are settled by the two bars above.
